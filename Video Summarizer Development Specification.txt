# Video Summarizer Development Specification

## 1. Project Goal

Build a **locally runnable video summarization system** that is stable, simple to develop, and easily extensible.

### Inputs

* YouTube URL
* Local video file

### Outputs

* Markdown summary
* Section summaries
* Timeline summary
* Structured outline summary

### Requirements

The system must support:

* Local LLM inference
* API-based LLM inference
* Easy model switching
* Custom summarization strategies
* Configurable summary depth

Supported models:

Local

* Ollama models

API

* Kimi
* DeepSeek

The system must prioritize:

* reliability
* modular design
* low hallucination risk
* simple development workflow

---

# 2. System Architecture

The system uses a **simple linear processing pipeline**.

```
Video
 ↓
yt-dlp (download)
 ↓
ffmpeg (audio extraction)
 ↓
WhisperX (speech-to-text)
 ↓
Transcript JSON
 ↓
Text Chunker
 ↓
LLM Adapter
 ↓
Map Summaries
 ↓
Reduce Summary
 ↓
Markdown Output
```

### Design Principles

Each stage must be:

* independent
* replaceable
* file-based

Intermediate files ensure:

* debugging
* restart capability
* reliability

# 10. LLM Adapter Layer

File:

```
llm/adapter.py
```

Purpose:

Provide a **unified interface** for different LLM providers.

Required interface:

```
generate(prompt) -> text
```

Example structure:

```
class LLMAdapter:

    def __init__(self, provider):
        self.provider = provider

    def generate(self, prompt):
        if self.provider == "ollama":
            return ollama_generate(prompt)

        if self.provider == "kimi":
            return kimi_generate(prompt)

        if self.provider == "deepseek":
            return deepseek_generate(prompt)
```

Supported providers:

```
ollama
kimi
deepseek
```

Adding a new model only requires creating a new client file.

# 11. Summarization Strategies

The system must support multiple summarization modes.

All strategies are located in:

```
summarizers/
```

---

## 11.1 Map-Reduce Summary

File:

```
summarizers/map_reduce.py
```

Process:

```
chunk → partial summary
partial summaries → final summary
```

Advantages:

* stable
* scalable
* low hallucination risk

Recommended for long videos.

---

## 11.2 Timeline Summary

File:

```
summarizers/timeline_summary.py
```

Output example:

```
00:00 Introduction
02:15 Core idea explained
05:40 Demonstration
08:30 Conclusion
```

Best for:

* tutorials
* lectures

---

## 11.3 Outline Summary

File:

```
summarizers/outline_summary.py
```

Output example:

```
1. Introduction
   1.1 Background
   1.2 Objective

2. Core Concepts
   2.1 Key Idea
   2.2 Implementation
```

Best for:

* educational content
* technical talks

---

# 12. Prompt Design

All prompts must follow strict rules.

Baseline prompt:

```
You must summarize ONLY the provided transcript.

Do not add external knowledge.

Do not speculate.

Use clear bullet points.
```

Purpose:

* reduce hallucination
* ensure factual accuracy

# 14. CLI Usage

Run with URL:

```
python main.py --url VIDEO_URL
```

Run with local file:

```
python main.py --file video.mp4
```

Select LLM provider:

```
--llm ollama
--llm kimi
--llm deepseek
```

Select summarization mode:

```
--mode outline
--mode timeline
--mode mapreduce
```

Optional parameters:

```
--detail-level 1-5
--bullet-count 10
```
